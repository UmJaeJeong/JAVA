해쉬코드 사용 목적: 논리적 동등객체는 HashCode의 값도 같아야한다.
		   (Hash 시리즈에 한해서만)

생성자 => 원복을 할때 많이 사용

얕은 복제(thin clone) : 원본 객체의 필드 값과 동일한 값을 가지는 
			새로운 객체를 생성하는것 
			ex) 기본형타입은 복사가 된다. 예를들어 클래스타입, 배열타입을 제외한!! 나머지 int, char....
			    

깊은 복제(deep clone) : 참조하고 있는 객체도 복제
			=> 위와같이 하기위해서 Object클래스의 clone()을 사용해야한다.
			ex) 참조타입을 가지고 있지 않는 클래스는 현업에 거의 없다.
			    클래스타입, 배열타입과 같이 참조변수일 경우에는 clone()함수를 Overriding을 해줘야한다.
			    참조객체를 복제하는 코드를 직접 넣어줘야한다.
			    (복제된것마다 독립된 객체를 만들기 위해서)
